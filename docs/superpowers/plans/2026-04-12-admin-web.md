# 广告商后台实现计划

> **面向 AI 代理的工作者：** 使用 superpowers:subagent-driven-development 逐任务实现。每个任务产出独立可测试的前端页面。

**目标：** 构建广告商使用的管理后台，包含工单看板、全流程管理、人员配置等功能

**架构：** Vue3 + Vite + Vue Router + Pinia + Element Plus + Axios，API 封装在 `src/api/`，页面组件在 `src/views/`

**技术栈：** Vue 3 Composition API, Element Plus, Vue Router, Pinia, Axios, ECharts, VueUse

---

## 项目初始化

- [ ] **步骤 1：搭建项目骨架**

```bash
npm create vite@latest admin-web -- --template vue
cd admin-web
npm install vue-router pinia axios element-plus @element-plus/icons-vue echarts @vueuse/core
npm install -D sass
```

创建目录结构：
```
admin-web/src/
├── api/              # API 封装（按模块分文件）
├── assets/           # 静态资源
├── components/       # 公共组件
│   ├── Layout/       # 布局组件
│   └── common/       # 通用组件
├── router/           # 路由配置
├── store/            # Pinia store
├── views/            # 页面组件
│   ├── dashboard/    # 看板
│   ├── work-orders/  # 工单管理
│   ├── departments/  # 部门管理
│   ├── clients/      # 甲方管理
│   ├── settings/     # 系统设置
│   └── login/        # 登录
├── utils/            # 工具函数
├── App.vue
└── main.js
```

- [ ] **步骤 2：配置 vite.config.js**

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: { alias: { '@': resolve(__dirname, 'src') } },
  server: { port: 5173, proxy: { '/api': { target: 'http://localhost:3000', changeOrigin: true } } },
})
```

- [ ] **步骤 3：配置 Element Plus 和 main.js**

- [ ] **步骤 4：创建登录页面 + 路由守卫**

登录页：居中卡片，logo、用户名、密码、登录按钮。表单验证。登录成功存 token 到 localStorage，跳转 /dashboard。

路由守卫：未登录跳转 /login，已登录访问 /login 跳转 /dashboard。

- [ ] **步骤 5：Commit**

**各环节操作说明：**

**申报信息（只读）：** 显示甲方提交的申报内容，照片以画廊形式展示

**派单信息：** 显示已分配的测量员，如无则显示 [分配测量员] 按钮，弹窗选择人员 + 截止日期

**测量数据：** 测量数据表格（长×宽×高等），现场照片画廊，测量员签名图片。操作：[审核通过] [驳回重测]（弹窗填写意见）

**设计：** 设计图片预览（大图查看器），内部审核 [通过] [驳回]，甲方确认状态显示

**生产：** 物料清单表格，排产日期选择，进度状态切换（排产→生产中→完成→发货），质检记录

**施工：** 施工记录时间线（照片+时间+备注），问题反馈列表，验收状态，甲方签名

**费用：** 报价金额 + 明细表格，收款记录（预付款/进度款/尾款），发票文件列表

**归档：** 归档状态，[生成归档报告] 按钮，归档文件打包下载

**售后：** 售后申请列表，处理记录，满意度评分
