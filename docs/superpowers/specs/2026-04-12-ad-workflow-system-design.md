# 广告工程全流程管理系统 - 设计文档

> 日期: 2026-04-12
> 状态: 已确认，等待实施

## 项目概述

一套 SaaS 化的广告工程全流程管理系统，支持多广告商（租户）独立使用，平台方（你）拥有全局数据监管能力。

## 技术栈

| 端 | 技术 | 说明 |
|----|------|------|
| 后端 API | Express.js + MySQL + Sequelize | 多租户逻辑隔离（tenant_id） |
| 甲方 Web 端 | Vue3 + Vite + Element Plus | PC 浏览器 |
| 甲方小程序 | 微信原生 + 微信云开发 | 轻量申报、扫码、通知 |
| 测量 APP | Flutter | iOS + Android 一套代码 |
| 广告商后台 | Vue3 + Vite + Element Plus | PC 管理后台 |
| 超级管理后台 | Vue3 + Vite + Element Plus | 平台方专用 |

## 多租户架构

- 所有表包含 `tenant_id` 字段区分租户（广告商）
- 超级管理后台查询时忽略 `tenant_id` 过滤
- 各租户数据通过中间件自动注入 `tenant_id`，防止越权
- 文件存储按 `tenant_id/` 目录隔离

## 系统角色

| 角色 | 所属端 | 说明 |
|------|--------|------|
| 平台管理员 | 超级后台 | 你，全局监管 |
| 广告商管理员 | 广告商后台 | 租户管理员，管理本公司所有业务 |
| 广告商派单员 | 广告商后台 | 负责工单分配 |
| 测量员 | APP + 广告商后台 | 现场测量 |
| 设计师 | 广告商后台 | 出设计稿 |
| 生产管理员 | 广告商后台 | 工厂生产跟踪 |
| 施工队长 | APP + 广告商后台 | 现场施工管理 |
| 财务人员 | 广告商后台 | 费用管理 |
| 甲方业务员 | 甲方 Web/小程序 | 提交申报 |
| 甲方领导 | 甲方 Web/小程序 | 审批申报单 |

## 工单流转（8 环节，严格顺序）

```
申报测量 → 甲方审批 → 内部派单 → 现场测量 → 设计 → 工厂生产 → 施工安装 → 费用管理 → 归档 → 售后维保
```

每个环节有明确的状态机：

| 环节 | 状态枚举 |
|------|----------|
| 申报测量 | `draft`（草稿）, `submitted`（已提交）, `rejected`（驳回） |
| 甲方审批 | `pending_approval`（待审批）, `approved`（已通过）, `rejected`（已驳回） |
| 内部派单 | `received`（已接收）, `assigned`（已派单） |
| 现场测量 | `measuring`（测量中）, `measured`（已完成）, `rejected`（需重测） |
| 设计 | `designing`（设计中）, `reviewing`（内部审核）, `approved`（设计通过）, `rejected`（需修改）, `client_confirmed`（甲方确认） |
| 工厂生产 | `scheduled`（已排产）, `producing`（生产中）, `completed`（生产完成）, `shipped`（已发货） |
| 施工安装 | `scheduled`（待施工）, `installing`（施工中）, `completed`（已完工）, `accepted`（已验收） |
| 费用管理 | `quoting`（报价中）, `quoted`（已报价）, `paid`（已收款）, `invoiced`（已开票） |
| 归档 | `archiving`（整理中）, `archived`（已归档） |
| 售后维保 | `pending`（待处理）, `processing`（处理中）, `resolved`（已解决）, `closed`（已关闭） |

## 模块一：甲方申报端（Web + 小程序）

### 1.1 组织架构

- 扁平两级：部门 → 人员
- 部门可配置管辖区域（省→市→区→街道四级）
- 人员归属部门，可关联多个管辖区域
- 角色：业务员（填报申报单）、领导（审批申报单）

### 1.2 申报管理

**新建申报单：**
- 项目基本信息：名称、地址（四级地址选择器）、项目类型、需求描述
- 现场照片上传（小程序支持拍照直传）
- 附件上传（参考图、合同扫描件等）
- 选择审批人（本部门领导）

**申报单列表：**
- 状态筛选（草稿/审批中/已通过/已驳回/流转中）
- 按部门/日期/状态筛选

**申报单详情：**
- 查看完整工单进度（8 环节当前状态）
- 查看各环节产出（测量数据、设计稿、施工照片等）
- 驳回申报单（审批环节）

### 1.3 消息通知

- 微信模板消息（小程序）：状态变更、审批待办、验收提醒
- Web 站内信：同上
- 通知类型：审批待办、工单进度更新、验收请求、售后反馈

## 模块二：工单主系统后台（广告商使用）

### 2.1 工单看板

- 各阶段工单数量卡片（申报/测量/设计/生产/施工/归档/售后）
- 超时工单预警（每个环节可配置超时天数）
- 按日/周/月统计图表
- 快捷操作：新建工单（广告商也可手动创建）、分配任务

### 2.2 申报接收

- 接收甲方审批通过的申报单（按区域自动匹配）
- 手动创建工单（广告商主动发起）
- 工单列表：支持按状态、甲方、区域、时间筛选
- 工单详情页：时间轴展示全流程进度

### 2.3 派单管理

- 待派单列表：已接收但未分配的工单
- 分配测量员：选择人员 + 设定截止时间
- 派单记录：历史派单追踪

### 2.4 现场测量

- 查看测量任务列表
- 审核测量结果（通过/驳回重测）
- 后台补录数据（APP 未覆盖的场景）

### 2.5 设计管理

- 设计师工作台：接收测量完成的工单
- 上传设计文件（CAD、效果图、3D 模型）
- 内部审核流程（设计主管审核）
- 甲方确认设计稿（通过后进入生产）

### 2.6 工厂生产

- 生产清单：根据设计稿自动生成物料清单
- 排产管理：设定生产计划
- 进度跟踪：已排产→生产中→完成→发货
- 质检记录
- 出库管理：关联施工工单

### 2.7 施工安装

- 施工队工作台：接收已发货工单
- 施工记录：照片、视频、备注
- 问题反馈：施工中遇到问题可上报
- 验收管理：甲方验收签字（APP 端电子签名）

### 2.8 费用管理

- 报价单：根据设计清单生成报价
- 收款记录：分阶段收款（预付款/进度款/尾款）
- 发票管理：开票记录、发票文件上传
- 费用统计：按工单/月度/年度汇总

### 2.9 归档管理

- 自动汇总：全流程文件自动归集
- 归档报告生成（PDF）
- 归档检索：按项目/甲方/日期搜索
- 文件下载/导出

### 2.10 售后维保

- 甲方提交售后申请（Web/小程序）
- 售后工单列表：待处理/处理中/已解决
- 处理记录：维修照片、说明
- 满意度评价

### 2.11 租户内部管理

- 部门管理：创建/编辑/删除部门 + 配置管辖区域
- 人员管理：添加员工、分配角色、绑定部门
- 角色权限：管理员/派单员/测量员/设计师/施工队长/财务
- 地址库管理：四级地址数据维护

## API 设计规范

- RESTful 风格，前缀 `/api/v1/`
- 多租户中间件自动注入 `tenant_id`
- 认证：JWT（广告商/甲方分开）
- 文件上传：`POST /api/v1/files`，返回文件 URL
- 分页：`?page=1&limit=20`
- 超级管理后台接口前缀 `/api/v1/admin/`，无 tenant_id 过滤

## 已确认技术决策

1. **地址库**：使用国家统计局标准地址库（省→市→区→街道四级），初始化时批量导入，后续手动维护增量更新
2. **电子签名**：Canvas 手写板，签名存为 PNG 图片关联到工单，测量和施工验收共用同一组件
3. **测量 APP 离线功能**：支持。离线时数据存 SQLite 本地缓存，网络恢复后自动同步。冲突策略：以服务端时间戳为准
4. **设计文件**：效果图 JPG/PNG（在线预览），源文件 CDR（下载用 CorelDRAW 打开）
5. **工单编号**：自定义规则（如 `GG-{year}-{seq}`），租户级别可配置，系统保证唯一性
6. **工单只能由甲方创建**，广告商可补录线下客户
7. **甲方审批开关可配置**
8. **推送到广告商后不可撤回**，需与广告商协商

## 已确认业务规则

### 工单流转规则
- 工单只能由甲方创建
- 甲方审批开关可配置（小客户可跳过审批直接推送广告商）
- 申报推送到广告商后不可撤回
- 工单分类：项目分类（520/国庆/日常等，字典可配置）+ 广告商内部判断广告类型

### 测量环节
- 树形结构采集：基本信息 → 多种材料 → 每种材料多个面 → 每面尺寸+照片
- 每面字段：尺寸（宽/高/厚等）、现场照片、备注、特别处理标记
- 自动计算面积（宽×高）
- 支持现场草图手绘
- 施工条件标记（高空作业/电源/夜间/车辆/物业/占道）
- 照片标注（圈画箭头文字）
- 支持精确到面的驳回（重测指定面）
- 测量报告 PDF 导出
- 历史工单数据参考
- APP 离线缓存 + 自动同步

### 设计环节
- 设计师工作台：左侧测量数据参考，中间上传设计图，右侧材料清单联动确认
- 上传效果图（JPG/PNG）+ 源文件（CDR），分开区域
- 效果图可关联一个面、多个面、或自定义名称（整体画面）
- 材料清单：测量量 → 设计师确认实际用量（可调），从材料库选材料
- 内部审核（设计主管审核通过/打回），不需要甲方确认
- 内部批注（仅内部可见）
- 修改意见时间线
- PDF 生成（效果图+测量数据+材料清单）
- 设计模板（常做项目存模板，下次套用）
- 甲方只读查看（效果图出后通知甲方）
- 审核通过后自动生成生产清单进入生产环节

### 生产环节
- 工单和生产任务解耦：相同材料+规格合并为一个生产任务统一生产
- 待排产材料汇总视图（按材料+规格汇总）
- 合并创建生产任务
- 每项材料独立状态：待排产→已排产→备料中→制作中→待质检→合格→入库
- 批次质检
- 问题上报（通知设计师）
- 生产文件管理（下载关联工单 CDR，工厂可上传合并排版 CDR）
- 第一版不做库存管理

### 领料施工环节
- 施工队到工厂领料，领料登记（支持批量领多个工单）
- 随车清单打印
- 现场异常上报
- 安装记录（APP 端：施工前/中/后照片）
- 余料退回登记
- 内部验收（施工主管审核照片、设计对比）→ 甲方验收（电子签名）
- 甲方验收自动通过天数可配置

### 费用结算环节
- 预算制：甲方（总公司）告知预算额度，广告商在额度内自由使用
- 报价关联预算，超预算预警
- 按项目批次结算
- 结算资料完整性检查（无客户签名不可提交）
- 结算报告 PDF 生成 + 电子签章
- 结算驳回记录（广告商承担损失）
- 驳回补资料重新提交

### 归档环节
- 验收通过后系统自动归档
- 自动汇总全流程文件
- 支持归档报告 PDF 导出
- 支持打包下载全流程文件（ZIP）

### 结算模板导出
- 上传甲方给的 Excel 结算模板
- 配置列映射（甲方列名 → 系统字段）
- 批量填充导出（选模板 + 选工单 → 导出 Excel）
- 按甲方模板管理，不同甲方用不同模板

### 售后维保
- 甲方提交售后申请（描述+照片）
- 广告商指派处理
- 处理记录（照片+说明）
- 甲方 1-5 星评价
- 关闭工单

### 统一组件
- 电子签名：Canvas 手写板，测量和施工验收共用
- 公共组件：图片上传/预览、地址级联选择、状态标签、文件列表

### 权限体系
- 基于角色的权限控制（RBAC），角色固定为：管理员、派单员、测量员/施工员、设计师、生产管理员、财务
- 每个角色有固定的菜单可见和操作权限（见下方权限矩阵）
- 第一版角色固定不可自定义，后期扩展自定义权限
- 平台操作日志与租户操作日志完全隔离，广告商看不到平台操作

### 设计变更联动
- 设计确认后材料清单可被设计师修改（如损耗、替换材料）
- 变更原因必填，变更记录留痕
- 变更自动同步到生产清单，工厂按确认后的量排料

### 多甲方多预算
- 一个租户服务多个甲方，每个甲方有独立预算
- 报价时选择关联甲方和预算，预算余额独立扣减
- 预算超限预警

### 数据备份与导出
- 支持按租户导出全量数据（Excel/CSV）
- 平台方定期自动备份数据库
- 工单列表支持导出 Excel
- 文件存储预留 OSS 切换能力（配置项：本地/OSS）

### 通知渠道
- 站内信（Web 端）
- 微信模板消息（小程序）
- 短信（预留，测量员可能不用微信）
- APP 推送通知（测量 APP 推送新任务）

### 工单看板视图
- 广告商后台支持看板视图（Kanban），按环节分列展示工单卡片
- 拖拽工单切换环节（仅同租户内操作）
- 快速查看工单状态
